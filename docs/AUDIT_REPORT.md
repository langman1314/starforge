## 四、无文案化原则确认

**Stage 4 已确认第一章骨架不提供：**
- 正文示例句
- 完整系统提示原文
- 完整聊天频道对话
- 开头示例句
- 结尾示例句

**骨架只规定：**
- **要什么**（作者意向、章节目标、情绪曲线）
- **不要什么**（禁止清单、信息释放边界、反模板系统文约束）
- **怎么承接**（一二章承接原则、收益设计、回收预埋）

正文写手必须基于本骨架自行生成表达，不得照抄骨架语言。

**后续 Stage 5 也必须延续无文案化原则：** 只能进一步细化段落功能、冲突推进、情绪节奏和信息边界，不得生成正文句子。

## 五、前期收益原则确认

**第一章：只发现异常，不完成回收。**
- 主角发现黑色废铁 → 触发文明回收系统异常识别 → 确认不属于普通求生资源 → 获得未完成动作（是否回收？）→ 读者产生追问

**第二章：可以小规模验证异常，但只能得到小而关键的收益。**
- 回收动作 → 回收风险 → 回收收益未知 → 回收不能直接暴富 → 回收可能只得到少量可用残余 → 回收后仍需普通资源配合 → 回收结果能帮助第一夜但不能彻底解决第一夜 → 回收结果会暴露更大未知

**前期采用"险胜式生存"原则：**
- 物质收益克制，认知收益放大
- 生存结果险胜，长期期待拉高
- 每次收益只能让主角领先普通人半步
- 每次收益只能解决一个眼前危机
- 每次收益都会留下新的缺口
- 读者爽点来自主角用别人忽略的东西活下来，而非暴富或碾压

**第一章 / 第二章不得出现：**
- 暴富
- 完整装备
- 高级科技
- 正式供暖核心
- 正式净水核心
- 榜一
- 追随者
- 余火号明确信息
- 燧明AI人格化登场
